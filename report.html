Hello! My name is Nguyen Ngoc Quoc Cuong, and a student at the University of London Computer Science (AI and Machine Learning)

Introduction

For the theme of my website, I decided to focus on a very close friend group that I have known for years. The website is meant to tell visitors stories about this group of friends’ life experiences. Therefore, to make this website capture the energy my group usually has and as if the readers were a part of the journey, I have to pay close attention to the website design choices.

Figure 1: The Gamers Unite

Things like “how interactive do I want this page to be?” or “how aesthetically pleasing is the website?” are important, but I find the “vibe” of a website is equally important. I actually have gone through many more design choices because I was conflicted about whether I should make a parasocial relationship website, or make it as transparent as possible. After a while, I decided on making this website feel as if I’m talking to an acquaintance. Two of the design choices I implemented to achieve this “feel” are randomised background pictures, and odd choices of words to make jokes.

  • The randomised background images javascript was implemented to the main page( index.html ) and the articles page( recentHangouts.html ) to show a sneak peek of the history my friends and I have together. The background pictures include the different time periods in our lives, random things that caught our interest, and a picture referencing “The Caretaker - It’s just a burning memory”. The album logo, albeit it is an extreme way to represent the message, is to show my fear of forgetting these precious memories I made with my friends when we all grow up and have our own families and responsibilities.

  • The odd choices of words, which is most prevalent in the article “Four Diners, Two Carbs” and some gifs’ / images’ descriptions, are meant to be like a window of how we have fun with one another. However, in this way of writing paragraphs for the website, it will look unprofessional. We gaslight each other, over-exaggerate things, and simply make references to our past mistakes or funny stories we experienced. Therefore, I wrote the website’s articles in a more toned-down and friendly way so that new acquaintances won’t have the wrong impression about us ,yet enough to still be able to know the 'vibe' that we are just a group of teenagers who want to have fun together (real). Hence, you can see it in the four atricles "Four Diners, Two Carbs", "1st Malaysia Trip", "Trampoline Frenzy", and "AFA".

Accessibility

A website that is accessible means that users with challenges can traverse the website easily and understand its content.
Thus, here are some of the features that make my website accessible.

  • Uses of high-contrast color

    • This is prevalent throughout the whole website. An example would be the home page’s “About Us” and “Member Introduction” Where all of the white color texts have black contrast color backgrounds.

    Another example would be the article title. When you opened an article, the banner image is always darkened, so that the white color title can be read.


  • Provide Alt Text for Images/Gifs/Videos/button

    • If you look through all of the HTML files, you can see that all media (except recentHangouts.html's temporary articles) have some form of alt text.

    This will ensure that users who can’t load the image know what it is, and allow users with visual impairment to understand the content when using assistive technologies such as screen reader.


  • Structuring my websites with appropriate tags

    • I make my website “semantically rich” by using appropriate tags at suitable sections. Throughout my website, i used tags like p, img, h1, ul, and table This will allow users who use screen readers to jump between tags and navigate the websites.

However, that does not mean my website is 100% accessible. I will need to improve it by implementing aria-labels to my website, providing transcripts of videos that have my friends talking to each other, and make modifications to my website so that users could use keyboard to navigate easier . This can be seen on the recentHangouts.html where with just a keyboard, you can’t view other articles from different years.

Usability

The website is meant to be a medium to introduce my close friend group. This can be seen through articles, group images, and photographs of our interest.

And since the usability of a website refers to how easily a user interacts with it. Thus, here are 3 examples of functions that I implemented to improve the users' experience in traversing the website.

  1. Section-jump function for long pages on the “1st Malaysia trip” article and the recentHangouts.html

    2. I find that my Malaysia trip article can be too long. Thus, I implement a function that allows users to skip to a specific section of the article. It is placed at the side and with little use of animation to prevent it from being too distracting for the user.

    The same can be said with the recenthangouts.html. Where I added a function at the side so the user can switch between years to view when different articles made from different years instead of compiling all past and present articles on the same page.


  2. Provide clear indications on the index.html

    3. This can be seen with the “GIF banner” and the “Member Introduction”. The “GIF banner” has arrows with grey backgrounds on both sides to allow users to know that they can click on it to display a different GIF. The “Member Introduction” section, for the member’s profile picture, has ‘cursor: pointer’ in the index.css so the users know that they can select the profile picture to display a different member description below it.


  3. Ease access to details on the recentHangouts.html

    4. Since there are so many articles, it is important to log the submitted date of them. However, adding dates to the descriptions and making them visible can make the website feel claustrophobic. Thus, I made that when the users hover over an article, it will expand and show the date of when the article was submitted.

Learning

This project not only allowed me to experience the joy of coding and to build something that I can look back to. This project also encourages me to do self-study to achieve and code certain functionality I mentioned in my wireframe. Here are the 3 most useful things I learned to during this project:

  • The image Slideshow

    • Though it is just an image slide show from the name, it is much more than that for me. The very same slideshow javascript concept can be applied throughout my whole website.
    When I was trying to implement the functionality of image slideshow based on https://www.w3schools.com/howto/howto_js_slideshow.asp , I find that the javascript section was the most confusing. The javascript section is quite different from javascript I learn from Introduction to Programming I. Thus, I tried to break it down to understand the concept behind it. And I felt exhilarated when I understood it.

    The line ‘let slides = document.getElementsByClassName("mySlides");’ means that it will turn the variable ‘slides’ into an array with objects inside it.

    In my case, ‘let intro = document.getElementsByClassName("member_Description");’ will create an array of ‘intro = [ {adipInstaPost:...,memberLayout:...} , {cuongInstapost:...,memberLayout:...} , … ].

    After understanding it, I applied a similar concept to the ‘Member Introduction’ section, the recentHangouts.html webpage, and the ‘Four Diners, Two Carbs’ dishes’ recipe section.


  • Learning how ‘:hover’ works fully

    • Though this is a piece of basic knowledge, just implementing this by itself can be make my website interactive but there is a catch. Shown in Figure 5 below, I find making things expand by hovering is cool but to have it snap back to place is jarring. Additionally in Figure 7, ‘:hover’ by itself only changes the element stated before it.

    Thus, to make my website more aesthetically, i delve deeper into how ‘:hover’ works. Long story short, i learned about ‘:not(:hover)’ and ‘:hover [element]’.
    By using ‘:not(:hover)’ in Figure 6, i can add ‘transition: 03s;’ so the attached element can slowly shrink back to its original size than snapping back.

    By using (example ‘.article:hover .date{‘ ) in Figure 8,I can change the ‘.date’ element when hovering over the ‘.article’ element.


  • 'margin: 0 auto;'

    • I prefer using flex to position elements for my webpage. But when I have to position an element inside of another element where ‘justify-content: center’ and ‘align-items: center’ won’t work, I usually face the problem of being unable to make it responsively stays in the middle. After learning about this from a thread of stacks overflow, my mind was Absolutely blown by simply it is to make an element stay in the middle of another element without using flex, float, or grid.

    Infact, you can see me using this multiple times throughout every single CSS file of every webpage just because of how easy it is to implement.

Evaluation I

I think the implementation of the slideshow concept in other sections of the website is quite successful. This is so because every time it works, I start to understand how javascript in building a website works.
An example would be ‘dot[i].style.borderColor = "white"’. After messing around, I understood that ‘element.style.borderColor = “white”’ is the same as typing ‘border-colour: white’ in CSS.
